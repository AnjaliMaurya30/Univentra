# Univentra

Univentra is a production-style campus engagement platform built with React, Vite, TypeScript, Tailwind CSS, Framer Motion, Recharts, React Hook Form, Zod, and Supabase.

It includes:

- public landing, browse, and event preview flows
- student dashboard, events, clubs, leaderboard, notifications, profile, registrations, and certificates
- organizer/admin dashboard, event management, approvals, announcements, analytics, and QR attendance scanning
- Supabase Auth, Postgres, Storage, Realtime hooks, RLS-ready SQL, and seed data
- QR pass generation plus camera-based QR attendance scanning
- in-app certificate rendering, PDF generation, storage upload, and download

## Tech Stack

- Frontend: React 19 + Vite + TypeScript
- Styling: Tailwind CSS
- Routing: React Router
- Forms: React Hook Form + Zod
- Charts: Recharts
- Motion: Framer Motion
- Backend: Supabase Auth + Postgres + Storage + Realtime
- Utilities: date-fns, qrcode, html5-qrcode, html2canvas, jspdf

## Local Setup

1. Install dependencies:

```bash
npm install
```

2. Copy the environment file:

```bash
cp .env.example .env
```

3. Fill in:

- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_ANON_KEY`
- `VITE_ALLOWED_EMAIL_DOMAIN` if you want optional campus-domain guidance/restriction in auth

4. Start the app:

```bash
npm run dev
```

5. Production build:

```bash
npm run build
```

## Supabase Setup

1. Create a new Supabase project.
2. In the SQL editor, run:
   - `supabase/schema.sql`
   - `supabase/seed.sql`
3. In Authentication:
   - make sure Email auth is enabled
   - password sign-in should remain enabled
4. In Storage:
   - the schema file attempts to create the required buckets automatically
   - if your project policy blocks bucket creation through SQL, create them manually with the same names:
     - `avatars`
     - `club-assets`
     - `event-banners`
     - `certificates`
5. In Realtime:
   - enable Realtime on `notifications` if you want the live notification experience

## Storage Buckets

The app expects four buckets:

- `avatars`
  - intended for public user avatar URLs
  - app uploads avatar files under `<user-id>/...`
- `club-assets`
  - intended for public logos and club media
- `event-banners`
  - intended for public event hero images
- `certificates`
  - intended to stay private
  - certificate PDFs are uploaded under `<user-id>/<event-id>/...`
  - downloads use signed URLs

`supabase/schema.sql` includes storage object policies for these buckets.

## QR Attendance Flow

Student side:

1. Register for an event.
2. A unique `qr_code_value` is generated by the database trigger or local demo backend.
3. The app renders a QR pass in event details and My Registrations.

Organizer/Admin side:

1. Open `/admin/attendance`
2. Select the event you are checking in attendees for
3. Use the device camera scanner
4. On a successful scan:
   - the QR payload is validated
   - the registration is looked up
   - attendance is marked as `attended`
   - `checked_in_at` is set
   - XP is awarded
   - notifications and badge checks run

## Certificate Flow

Eligibility rules:

- registration status must be `attended`
- event status must be `completed`
- `certificate_enabled = true`

Flow:

1. Completed attended registrations become certificate-eligible through database functions.
2. An organizer/admin opens the event details screen for the event.
3. They issue a certificate from the attendee list.
4. The app renders the certificate template, generates a PDF, uploads it to the `certificates` bucket, and stores/updates the certificate record.
5. Students download issued certificates from Certificate Center.
